# Revisi Klien — 18 Juli 2026

Catatan revisi klien (chat + foto spec "F. Data Prestasi Atlet"). Setiap butir
dipetakan ke spec yang terdampak.

| # | Catatan | Keputusan | Spec terdampak |
|---|---|---|---|
| 1 | Admin DISPORA sebagai tipe user baru | Role `ADMIN_DISPORA` — **hanya melihat data dan mencetak laporan** (read-only lintas cabor: dashboard, atlet, pelatih, prestasi, monitoring, event, cabor, pelaporan). Tidak ada akses tulis; guard API menolak semua mutasi. Konstanta baru `OVERSIGHT_ROLES` / `UNSCOPED_VIEW_ROLES` | `001-auth-rbac` |
| 2 | Status atlet **Pensiun** | Enum `AthleteStatus` + `RETIRED` (label "Pensiun"); dapat dipilih langsung di form atlet | `004-atlet` |
| 3 | Status **Cedera** butuh input tanggal + keterangan | Field `Atlet.tanggalCedera` + `Atlet.keteranganCedera`; muncul di form saat status Cedera, tampil di Biodata; otomatis dikosongkan saat status berubah dari Cedera (termasuk via monitoring STATUS_CHANGE) | `004-atlet`, `008-monitoring-atlet` |
| 4 | Pelatih: file lisensi **PDF/JPG** + tanggal terbit & kedaluwarsa | `Pelatih.lisensiFileUrl` + `POST /pelatih/:id/lisensi` (filter mimetype pdf/jpeg); tanggal memakai field lama `masaBerlakuMulai`/`masaBerlakuAkhir` yang sudah ada di form | `005-pelatih` |
| 5 | Tingkatan lisensi pelatih: **Nasional** dan **Cabang Olahraga** | Dropdown tetap (DB tetap String; nilai lama dipertahankan di select) | `005-pelatih` |
| 6 | Tingkat kejuaraan prestasi (foto spec F): Olympic, Asian Games, SEA Games, PON, BK PON (Pra PON), Porwil, Porprov, Kejurnas, Kejurda | Enum `CompetitionLevel` diperluas (`COMPETITION_LEVEL_CHOICES`); nilai lama Kota/Provinsi/Nasional/Internasional jadi legacy (valid untuk data lama, tidak ditawarkan untuk data baru) | `007-prestasi-atlet` |
| 7 | Level **PON** dan **Olimpiade** untuk tingkat event & tingkat atlet | `EventLevel` dan `AthleteLevel` + `PON`, `OLIMPIADE` | `017-event-calendar`, `004-atlet` |
| 8 | Sertifikat prestasi atlet **bisa lebih dari satu** | Model baru `PrestasiSertifikat` (1-N); upload menambah (bukan mengganti); endpoint hapus per sertifikat; `Prestasi.sertifikatUrl` lama tetap tampil sebagai legacy | `007-prestasi-atlet` |
| 9 | Sertifikat prestasi tampil di data atlet, **tidak di halaman publik** | Tab Prestasi pada detail atlet menampilkan seluruh sertifikat; endpoint publik memakai `select` eksplisit tanpa URL sertifikat | `007-prestasi-atlet`, `018-public-pages` |
| 10 | Pendidikan terakhir atlet: **Pendidikan Tinggi, SMA/SMK/MA/MAK, SMP/MTs, SD/MI** | `EDUCATION_LEVELS` diganti; nilai lama (mis. "S1") dipertahankan di select saat edit | `004-atlet` |
| 11 | Jabatan pengurus cabor: dropdown **Ketua, Wakil, Sekretaris, Bendahara, Kepala Bidang** + **Lainnya** (input manual) | Dropdown di modal pengurus (DB tetap String) | `006-pengurus-cabor` |
| 12 | Superadmin dapat **aktif/nonaktifkan akun & cabor** | Akun: tombol Aktifkan/Nonaktifkan di daftar pengguna (`PATCH /users/:id` `isActive`). Cabor: field `CabangOlahraga.isActive` + `PATCH /cabor/:id/active` (SUPER_ADMIN saja) | `001-auth-rbac`, `003-cabang-olahraga` |
| 13 | Cabor dinonaktifkan → admin cabor tersebut ikut nonaktif | Transaksi: nonaktif cabor men-set `isActive=false` semua user `ADMIN_CABOR` cabor itu. Mengaktifkan kembali cabor **tidak** otomatis mengaktifkan akunnya (aktifkan per akun di menu Pengguna) | `001-auth-rbac`, `003-cabang-olahraga` |
| 14 | Menu **Pengaturan Sistem** untuk superadmin | Halaman hub `/settings` (SUPER_ADMIN saja) berisi tautan: kelola/buat pengguna, kelola/buat cabor, pengumuman, slider, riwayat aktivitas | `001-auth-rbac` |
